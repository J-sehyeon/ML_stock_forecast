# ML_stock_forecast
## Overview
이 연구의 핵심 주제는 지식그래프를 사용해 주가를 예측하는 것이다.<br>
팔란티어라는 기업은 온톨로지라는 개념을 사용해 기업, 국가, 국방에 도움을 주는 기업이다.<br>
온톨로지란 객체와 관계, 각각의 속성을 정의한 메타 정보이다. 이를 기반으로 KG: Knowledge Graph 가 생성된다.<br>

팔란티어가 온톨로지를 어떻게 서빙하는지 구체적인 기술이 나와있지 않아 똑같이 구현할 수는 없다.<br>
하지만 온톨로지라는 기술이 현실 세계의 관계를 디지털 세계로 복사하는 것이 가능하다는 사실을 알려줬기 때문에, 이 기술과 LLM, 실시간 주가 데이터를 결합한다면 주가의 단기성 변동을 예측할 수 있을 것이라 생각한다.<br>

## PoC
### Knowledge Graph
> 지식그래프의 생성을 LLM으로 자동화하는 것을 목표로 한다.
> 이를 위해서는 주가에 영향을 주는 요인들과 그 관계를 설명할 수 있는 정확한 온톨로지가 필요하다.
> 그리고 이 온톨로지를 기반으로 데이터를 수집하고 정제, 적재하는 agent가 필요하다. 이 때 이 agent의 정확도는 PoC의 중요한 체크포인트가 되겠다.

KG 생성의 PoC는 세 단계를 거친다.
#### 1. Crawling
> 목적 기업에 대해 주변 객체와 그 관계들을 검색하여 찾는다.

##### Problems
- 어떻게 찾을 것인가? : agent 사용 혹은 객체, 관계 사전을 통한 검색
- 출처는 어떻게 할 것인가?
- 특정 언론사에 의한 과대, 과소 해석 혹은 거짓뉴스를 어떻게 처리할 것인가?
<br> : 여러 언론사에서 같은 정보 전달 -> 진실로 판별 혹은 기업의 공식 입장

#### 2. Embedding
> 검색된 문장을 벡터 임베딩을 하고 문장을 속성을 포함한 triple로 만들어서 지식 그래프에 적재할 준비를 마친다. 

##### Problems
- 어떤 속성을 어떻게 추출할 것인가?
- 임베딩 모델은 무엇을 선택할 것인가?
- 복잡한 관계는 어떻게 처리할 것인가?
- UNK 객체, 관계 처리 : LLM에 입력하여 새로운 객체 혹은 관계로 정의할 필요 판단

#### 3. KG
> 지식 그래프 적재하기 전 똑같은 정보가 있는지, 스키마에 위배되는 triple이 있는지 확인 후 적재. 그리고 아직 채워지지 않은 관계 출력하여 리포트
